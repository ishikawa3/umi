# うみ — 拡張計画

実装者向け（Claude Sonnet 5想定）の作業計画。上から順にフェーズ単位で実装する。
各フェーズは独立してリリース可能。**1フェーズ完了ごとにビルド・ブラウザ確認・コミットする。**

---

## 0. 前提知識（実装前に必ず読む）

### プロジェクトの現状

- 公開URL: https://ishikawa3.github.io/umi/ （mainにpushすると自動デプロイ）
- 潮流フローフィールド1ページのみ。構成:
  - `src/api.ts` — 海しるAPIクライアント（429リトライ、JST時刻変換）
  - `src/field.ts` — 散在点からのIDW補間ベクトル場
  - `src/render.ts` — Canvas 2Dパーティクル描画（残像トレイル）
  - `src/landmask.ts` — 国土地理院淡色地図タイルによる水域判定
  - `src/main.ts` — UI・時間軸・メインループ
  - `src/config.ts` — APIキーと定数
- Vite + TypeScript + Canvas 2D。**フレームワーク・外部ライブラリは追加しない**（現状JS 14KB。この軽さは作品の一部）。

### デザイン言語（全ページ共通・厳守）

- 背景 `#04070f`、インク `rgba(226,238,248,…)`、アクセント `#7fe3e0`。トークンは `src/style.css` の `:root` を再利用
- 明朝体（Hiragino Mincho ProN系）、広いletter-spacing、小さめの文字
- 画面はほぼ全部をビジュアルに使い、UIは四隅に小さなHUDのみ。枠・カード・影は使わない
- データの色: 量の表現は**単一色相のシーケンシャルランプ**（潮流ページの群青→シアン→白を踏襲）。虹色・多色は禁止。カテゴリを色分けする場合も最大3色まで
- 全ページ左下に出典クレジット（海しる＋使うAPI名、地理院タイル使用時は「国土地理院」、「航海には使用しないでください」）
- ホバーで実データ値を出すツールチップ（`#tooltip` のスタイル再利用）

### 海しるAPIの使い方（検証済みの事実）

- ベース `https://api.msil.go.jp`、ヘッダー `Ocp-Apim-Subscription-Key: 0e83ad5d93214e04abf37c970c32b641`（`src/config.ts` にある試用キー）。CORS全開放、ブラウザから直接叩ける
- v2系はArcGIS REST形式:
  `GET /{path}/v2/MapServer/{layer}/query?f=geojson&geometry={"xmin":..,"ymin":..,"xmax":..,"ymax":..}&geometryType=esriGeometryEnvelope&inSR=4326&outFields=*`
- 1リクエスト最大1000件。超過時はレスポンス末尾に `"exceededTransferLimit":true` → `resultOffset` でページング
- 429が返ることがある → `src/api.ts` の `msilFetch` を必ず経由する（リトライ内蔵）
- 未知のAPIの層番号・パラメータは、ポータルの管理APIで機械的に調べられる:
  - 一覧: `https://portal.msil.go.jp/mapi/apis?api-version=2022-04-01-preview`
  - 操作と層enum: `https://portal.msil.go.jp/mapi/apis/{id}/operations?api-version=2022-04-01-preview`
- 詳細は `~/.claude/projects/…/memory/msil-api-notes.md` にもある

### 検証済みAPIデータ（このplan作成時に実測）

| API | エンドポイント | 中身 |
|---|---|---|
| 海洋速報 | `quick-bulletin/v2/MapServer/{6,8,9}/query` | 6=黒潮（MultiLineString）、8=対馬暖流、9=宗谷暖流。属性: `解析対象日` "R8(2026)/06/23", `速報発行日`, `速報号数`, `海流の種別`。`/MapServer/range` という時系列出力もある（パラメータは要調査） |
| 潮汐推算 | `tide-prediction/v3/station` → 313地点（GeoJSON Point, `stationCode`,`nameJa`,`nameEn`）、`/data?stationCode=1314&date=20260707` → `{time, interval:60, tide:[cm,…]}` 1分間隔1440個 | 東京=晴海 `1314` |
| 航行警報 | `navigational-warnings/v2/MapServer/{1,3}/query` | 1=Point、3=Polygon。属性: `name`（例 "06-260085 豊後水道、灯台消灯、仮灯設置"）、`description`（HTML混じりテキスト） |
| 灯台 | `lights/lighthouse/v2/MapServer/1/query` | Point。属性は `航路標識番号`,`名称`,`読み` のみ。**灯質（点滅パターン）は無い** |

## フェーズ1: マルチページ化（共通基盤）

**ゴール**: 現在の潮流ページを保ったまま、複数作品を持てるサイト構造にする。

1. Vite MPA化。`vite.config.ts` に rollup input を追加:
   `index.html`（潮流・現状のまま）, `kuroshio.html`, `tide.html`
   （フェーズ2・3で中身を作る。この時点では空のプレースホルダでよい）
2. 共通ナビ: 各ページ右下（凡例の上）に縦書き風の小さなリンク列を置く。
   `ながれ / くろしお / しおどき` — 現在ページは `--accent` 色、他は `--ink-3`。
   実装は `src/nav.ts` に `mountNav(current: string)` を作り各ページから呼ぶ
3. 共通化リファクタ: `src/api.ts` に v2 ArcGISクエリ用ヘルパー
   `arcgisQuery(path: string, layer: number, bbox, extraParams?)` を追加し、
   `fetchContours` をこれで書き直す
4. `src/style.css` はそのまま全ページで共有。ページ固有スタイルは増やさない

**確認**: `npm run build` が3ページ分成功し、既存の潮流ページがローカルで従来どおり動く。ナビのリンクで相互遷移できる（`base=/umi/` を壊さないよう相対リンク `./kuroshio.html` を使う）。

## フェーズ2: くろしお — 黒潮のうねり

**ゴール**: 日本列島の南岸をうねる黒潮流軸を1本の光る線として描く、静的だが呼吸するページ。3海流（黒潮・対馬暖流・宗谷暖流）を淡く重ねる。

**データ**:
- `quick-bulletin/v2/MapServer/6/query`（黒潮）、`8`（対馬暖流）、`9`（宗谷暖流）。bboxは日本全域 `{"xmin":122,"ymin":24,"xmax":148,"ymax":46}`
- 最新の解析だけ欲しい: まず `outFields=*` で取得し、`速報号数` が最大（最新）のfeatureだけ使う。1リクエストで複数号数が返る場合があるため必ずフィルタする
- `/MapServer/range` の時系列出力を調査し（operations APIでパラメータ確認）、
  **過去N回分の流軸が取れたら**時間スライダーで蛇行の変化を見せる。取れなければ最新のみで良い（スコープを膨らませない）

**ビジュアル仕様**:
- 日本全域の正距円筒投影（潮流ページの `fitProjection` と同じ式。共通化してもよい）
- 背景: 陸のシルエット。`landmask.ts` を流用し、日本全域を z6〜7 でマスク生成 →
  陸ピクセルを `rgba(226,238,248,0.05)` 程度で点描 or 塗り。等深線は使わない（広域では重い）
- 黒潮流軸: 線に沿ってパーティクルを流す。`field.ts` は使わず、ポリラインに沿った
  弧長パラメータで動かす単純な実装で良い（線上に等間隔で点を置き、上流→下流へ進め、
  端まで来たら先頭へ戻す）。色はアクセントのシアン系グラデーション、
  線自体も `rgba(127,227,224,0.15)` で細く敷く
- 対馬暖流・宗谷暖流は同じ仕組みで彩度を落として淡く
- 左上タイトル「黒潮」+ 英字 "Kuroshio Current"、その下に `解析対象日` と `速報号数` を小さく表示（データの鮮度が作品の一部）
- ホバー: 流軸の近く（画面距離10px以内）で海流名を表示

**確認**: ブラウザで黒潮の蛇行形状が視認でき、コンソールエラーなし。属性の日付が表示されている。スクリーンショットで陸シルエットと流軸の位置関係が正しい（黒潮が本州南岸沖にある）こと。

## フェーズ3: しおどき — 潮の呼吸

**ゴール**: 全国313の検潮地点が現在の潮位で明滅する日本地図＋選んだ地点の1日の潮位曲線。「日本列島が潮汐で呼吸している」ことを見せる。

**データ**:
- 地点一覧: `tide-prediction/v3/station`（313点、初回のみ取得しメモリ保持）
- 潮位: `tide-prediction/v3/data?stationCode={code}&date={YYYYMMDD}` → 1分間隔1440値(cm)
- **313地点×毎日を全部取るのは禁止**（リクエストが多すぎる）。方式:
  - 地図の明滅: 表示中の時刻の潮位を出すには全地点のデータが要るため、
    **地点を約60点に間引く**（緯度経度グリッドで代表点選択）。60リクエストを
    起動時に `Promise.all` で並列取得（msilFetch経由、429リトライに任せる）し、
    セッション内キャッシュ
  - 地点詳細: クリックした地点はその場で個別fetch
- 潮位の正規化: 地点ごとに当日のmin/maxで0..1に正規化した値で明滅させる
  （絶対値比較は意味がないため。凡例も「干潮←→満潮」のラベルにする）

**ビジュアル仕様**:
- 日本全域投影＋淡い陸シルエット（フェーズ2の部品を再利用）
- 各地点: 小さな円。満潮に近いほど明るく大きく（半径2〜5px、シアン→白）、
  干潮側は暗い群青。**現在時刻を自動で進める**（実時間1秒=実潮汐3分、潮流ページと同じ係数）
  ＋下部に潮流ページと同じ時刻スライダー（当日0時〜24時）
- クリックで地点選択 → 画面下1/4に潮位曲線を描く:
  1440点の折れ線（`--accent` 1.5px）、現在時刻の位置に光る点、
  満潮・干潮の極値にだけ小さく時刻ラベル（全点ラベルは付けない）。
  地点名を明朝体で大きく（タイトルと同格に）
- 曲線はCanvasでもSVGでも良いが、ページ全体のcanvasと重ねるならCanvasに統一

**確認**: 初期表示で60地点が明滅し、東京湾と九州で位相が違う（満ち引きのタイミングのずれ）ことが見える。任意の地点クリックで曲線が出て、ホバーで `潮位 xxx cm / hh:mm` が出る。コンソールエラーなし・429が出ても最終的に全点表示されること。

## フェーズ4: こえ — 海の掲示板（航行警報）※任意・余力があれば

**ゴール**: いま日本の海で起きていることば（航行警報）を、地図上の静かな点と一覧で見せる。

**データ**: `navigational-warnings/v2/MapServer/1/query`（Point）と `3`（Polygon）。日本全域bbox。`name` は「番号 海域、内容」形式、`description` はHTML混じり（**タグを除去してテキスト化**し、リンクは無視）。

**ビジュアル仕様**:
- フェーズ2の日本地図部品を再利用。警報Point/Polygon重心を琥珀色 `#e8c468` の点で置く
  （安全情報なのでアクセントのシアンと区別する。この1色のみ追加を許可）
- 右側に `name` を並べた縦のリスト（最新〜20件、明朝体、`--ink-2`）。
  リスト項目ホバー ↔ 地図上の点ハイライトを双方向連動
- クリックで `description` の整形テキストを表示（HUDパネル、背景 `--panel`）

**確認**: 実在の警報（例: 灯台消灯、海洋調査）が読める。点とリストの連動が動く。

## やらないこと（スコープ外・提案もしない）

- 灯台の点滅表現（灯質データがAPIに無い。位置と名前しか取れない）
- 3Dライブラリ・地図ライブラリ・UIフレームワークの導入
- サーバー/プロキシの追加（全APIがCORS開放済みのため不要）
- 漁業権・海底ケーブル等の追加レイヤー（今回の計画では扱わない。次期候補）

## 全フェーズ共通の完了条件

1. `npx tsc --noEmit` と `npm run build` が通る
2. ローカル（`npm run dev`）をブラウザで開き、スクリーンショットで見た目を確認
   （粒子・線が視認できる明るさか、HUDが重なっていないか、コンソールエラーがないか）
3. モバイル幅（<720px）で最低限崩れない（潮流ページ同様、サイドUIは隠して良い）
4. クレジット表記がある
5. コミットメッセージは日本語で、何のフェーズかを冒頭に書く。mainへpushするとデプロイされる
